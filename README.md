# 🔐 TEA Cipher FX

[![Java Version](https://img.shields.io/badge/Java-21%2B-ED8B00?logo=java)](https://oracle.com/java)
[![JavaFX](https://img.shields.io/badge/JavaFX-Modern-brightgreen?logo=java)](https://openjfx.io/)

**TEA Cipher FX** is a robust JavaFX-based desktop application that implements the canonical **Tiny Encryption Algorithm (TEA)**. This project is designed as a professional, secure, and user-friendly tool for local data encryption.

## 📌 Project Overview

The application provides reliable text encryption using a 128-bit key. Development is heavily focused on the precise implementation of the cryptographic primitive while maintaining a seamless user experience.

### ✨ Key Features
* **Algorithm:** Canonical TEA algorithm (Wheeler & Needham, 1994).
* **Endianness:** Little-Endian byte order implementation.
* **Padding:** PKCS#7 standard padding.
* **Security:** Built-in password complexity validation and secure key generator.
* **NIST Ready:** Ability to export ciphertext as raw bytes (.bin) and bitstreams (.txt) for statistical testing.
* **User Interface:** Modern JavaFX GUI featuring seamless Drag & Drop support.

## 🧬 Mathematical Model

The TEA algorithm is based on a Feistel network structure. It utilizes modular addition (modulo $2^{32}$), bitwise XOR operations, and bitwise shifts.

### The Magic Constant ($\delta$)
In each round, a constant $\delta$ (derived from the Golden Ratio) is used to prevent simple attacks based on the symmetry of the rounds:
$$\delta = \lfloor 2^{31} (\sqrt{5} - 1) \rfloor = \text{0x9E3779B9}$$

### Core Encryption Cycle
One full TEA cycle consists of two rounds. For a data block $v = \{v_0, v_1\}$ and a key $K = \{k_0, k_1, k_2, k_3\}$, the core operations are:

$$v_0 = v_0 + \left( ((v_1 \ll 4) + k_0) \oplus (v_1 + sum) \oplus ((v_1 \gg 5) + k_1) \right)$$
$$v_1 = v_1 + \left( ((v_0 \ll 4) + k_2) \oplus (v_0 + sum) \oplus ((v_0 \gg 5) + k_3) \right)$$

Where $sum$ is the accumulated value of $\delta$ for the current round. The algorithm executes 32 full cycles (64 rounds) per block.

## 📊 Statistical Testing (NIST SP 800-22)
To confirm cryptographic strength, the application supports data generation for the NIST Statistical Test Suite. Every encryption session automatically creates:
1. Binary Raw (.bin): Used for testing with utilities like dieharder.
2. ASCII Bitstream (.txt): A text sequence of '0' and '1' for online testers (e.g., [Random Bitstream Tester](https://mzsoltmolnar.github.io/random-bitstream-tester)).

## 📂 Workspace Structure

The application automatically organizes your cryptographic data by creating a dedicated directory hierarchy for each unique key session (based on the key's hash):

```text
📁 KeySession_[hash]/
 ├── 📄 key.op                     # Key file (can be dragged into the UI)
 ├── 📁 Encrypted/
 │    ├── 📄 ENCRYPTED_...txt      # Result in Base64 (for readability)
 │    └── 📄 NIST_BITS_...txt      # Bitstream (0/1) for testing
 └── 📁 Decrypted/                 
      └── 📄 DECRYPTED_...txt
```

## 🚀 Getting Started

### Prerequisites
* **JDK:** Java Development Kit 21 or higher.
* **Build Tool:** Apache Maven.

### Compilation (Maven)
To build the project from the source code, navigate to the project root and run:
```bash
mvn clean package
```
Upon successful execution, the compiled executable `.jar` file (containing all necessary dependencies) will be located in the `target/` directory.

### Running the Application
To launch the application from the `.jar` file without opening a console window, use `javaw`:
```bash
javaw -jar target/TEA-Cipher-FX-1.0-SNAPSHOT.jar
```
*Note: Ensure the file path matches the actual artifact name generated by your `pom.xml`.*

---

## 📦 Creating a Windows Executable (.exe)

To convert this Java application into a native standalone executable for Windows, you can use one of the following methods:

### Method 1: Using `jpackage` (Recommended, JDK 21+)
Run the following command in your terminal:
```bash
jpackage --type app-image --dest dist --name "TeaCipherOP" --input target --main-jar TEA-cipher-FX-1.0-SNAPSHOT.jar --main-class com.op.teacipherfx.Launcher
```

### Method 2: Using Launch4j
1. Download and open [Launch4j](https://launch4j.sourceforge.net/).
2. In the **Basic** tab, set the path to your `.jar` in the "Jar" field and your desired `.exe` name in the "Output file" field.
3. In the **JRE** tab, set the "Min JRE version" to `21`.
4. Build the wrapper.
